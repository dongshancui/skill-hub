Harness Engineering

工程实践方法论

基于 Claude Code Harness 的四阶段开发流程
让 AI 自动化承载 80% 的开发工作量

01
需求分析 & 概要设计
弄清楚要做什么，再想怎么做
🎯
明确业务场景
确定边界、目标、约束条件
→
🗂️
划分功能模块
按职责拆分，确定粒度
→
🏗️
概要设计
技术方案选型、架构决策
02
任务拆分 & 验收确认
产出可执行的任务文档，是后续自动化的核心输入
🤖
调用 task-planner
生成结构化任务树
→
✅
定义验收点
每个任务对应可验证标准
→
📋
人工确认审查
核查任务边界与完整性
📄 tasks.md  ·  任务清单与分组
☑️ checklist.md  ·  验收点清单
03
自动化开发循环
主会话持续调度 subagent，验证失败自动重试
⚡ 自动化覆盖 ~80%
📖
读取任务文档
主会话加载 tasks.md & checklist.md
→
🚀
调度 Subagent
按任务组并发或串行执行
→
🔍
验证验收点
逐条核查 checklist 通过率
🔄
验收失败 → 分析原因 → 调整策略 → 重新调度 subagent → 再次验证
直到 所有任务组验收点全部通过 才进入下一阶段
04
人工精修
处理自动化覆盖不到的细节与边界情况
👤 人工投入 ~20%
👁️
整体走查
发现自动化遗漏的细节问题
→
🔧
手动修复
针对性解决小问题
→
🎉
验证交付
回归测试，问题清零后上线
